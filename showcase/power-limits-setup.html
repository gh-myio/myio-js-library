✨ Features

Key capabilities of the Power Limits Setup Modal component.

📊 Configure 4 status thresholds: StandBy, Normal, Alert, Failure
🔌 Support for 9+ device types (Elevator, Motor, Chiller, etc.)
⚡ 11 telemetry types (consumption, voltage, current, etc.)
💾 Persists to ThingsBoard customer server_scope attributes
🔄 Partial JSON updates without data loss
✅ Form validation with inline error messages
🎨 Customizable styles and theming
⌨ Keyboard navigation and accessibility support

🏷 Device Status Types

The modal configures threshold limits for these device operational statuses.

⏸ StandBy ✅ Normal ⚠ Alert ❌ Failure

Each status has a Base Value (minimum threshold) and Top Value (maximum threshold). The system uses these ranges to classify device telemetry readings in real-time.

📄 JSON Schema (RFC-0078)

The modal manipulates the mapInstantaneousPower JSON structure stored in customer attributes.

{ "version": "1.0.0", "limitsByInstantaneoustPowerType": [ { "telemetryType": "consumption", "itemsByDeviceType": [ { "deviceType": "ELEVADOR", "name": "mapInstantaneousPowerElevador", "description": "Power limits for ELEVADOR - consumption", "limitsByDeviceStatus": [ { "deviceStatusName": "standBy", "limitsValues": { "baseValue": 0, "topValue": 0 } }, { "deviceStatusName": "normal", "limitsValues": { "baseValue": 1, "topValue": 10000000 } }, { "deviceStatusName": "alert", "limitsValues": { "baseValue": 10000001, "topValue": 10000002 } }, { "deviceStatusName": "failure", "limitsValues": { "baseValue": 10000003, "topValue": 10000004 } } ] } ] } ] }

💻 Code Examples

How to use the Power Limits Setup Modal in your application.

// Basic usage import { openPowerLimitsSetupModal } from 'myio-js-library'; const modal = await openPowerLimitsSetupModal({ token: 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...', customerId: 'a1b2c3d4-e5f6-7890-abcd-ef1234567890', tbBaseUrl: 'https://tb.myio-bas.com', onSave: (json) => console.log('Saved:', json), onClose: () => console.log('Modal closed') }); // Clean up when needed modal.destroy();
// Advanced usage with pre-selected device type and existing data const modal = await openPowerLimitsSetupModal({ token: jwtToken, customerId: 'customer-uuid', tbBaseUrl: 'https://tb.myio-bas.com', deviceType: 'MOTOR', telemetryType: 'total_current', existingMapPower: existingJsonData, styles: { primaryColor: '#6366f1', borderRadius: '16px' }, onSave: (updatedJson) => { console.log('Configuration saved!'); // Sync with backend, update UI, etc. } });

📖 API Reference

Parameters accepted by openPowerLimitsSetupModal().

Parameter Type Required Description
token string Yes JWT token for ThingsBoard authentication
customerId string Yes ThingsBoard customer UUID
tbBaseUrl string No ThingsBoard API base URL (defaults to current origin)
deviceType string No Pre-selected device type (default: 'ELEVADOR')
telemetryType string No Pre-selected telemetry type (default: 'consumption')
existingMapPower object No Existing configuration JSON to pre-populate form
onSave function No Callback after successful save with updated JSON
onClose function No Callback when modal closes
styles object No Custom style tokens (primaryColor, borderRadius, etc.)
