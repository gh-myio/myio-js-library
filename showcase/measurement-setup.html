✨ Features

Key capabilities of the Measurement Setup Modal component.

💧 Water: m³ or Liters with configurable decimals (0-6)
⚡ Energy: Auto, kWh, or MWh with force option
🌡 Temperature: Celsius or Fahrenheit
🔢 Configurable decimal places per domain
💾 Persists to ThingsBoard customer SERVER_SCOPE
👁 Live preview of formatted values
🇧🇷 Interface em Português (pt-BR)
🎨 Customizable styles and theming

🏷 Supported Domains

The modal configures display settings for these measurement domains.

💧 Water (m³ / L) ⚡ Energy (kWh / MWh) 🌡 Temperature (°C / °F)

📄 JSON Schema (RFC-0108)

The modal manipulates the measurementDisplaySettings JSON structure stored in customer attributes.

{ "version": "1.0.0", "updatedAt": "2025-12-23T12:00:00.000Z", "water": { "unit": "m3", "decimalPlaces": 3, "autoScale": true }, "energy": { "unit": "auto", "decimalPlaces": 3, "forceUnit": false }, "temperature": { "unit": "celsius", "decimalPlaces": 1 } }

💻 Code Examples

How to use the Measurement Setup Modal in your application.

// Basic usage import { openMeasurementSetupModal } from 'myio-js-library'; const modal = await openMeasurementSetupModal({ token: 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...', customerId: 'a1b2c3d4-e5f6-7890-abcd-ef1234567890', tbBaseUrl: 'https://tb.myio-bas.com', onSave: (settings) => { console.log('Settings saved:', settings); // Refresh dashboard to apply new settings }, onClose: () => console.log('Modal closed') }); // Clean up when needed modal.destroy();
// Using settings in your widget const settings = await fetchMeasurementSettings(customerId, token); // Format water value const waterValue = 1234.567; const unit = settings.water.unit === 'liters' ? 'L' : 'm³'; const display = settings.water.unit === 'liters' ? (waterValue * 1000).toFixed(settings.water.decimalPlaces) : waterValue.toFixed(settings.water.decimalPlaces); console.log(`${display} ${unit}`); // "1234567.000 L" or "1234.567 m³"

📖 API Reference

Parameters accepted by openMeasurementSetupModal().

Parameter Type Required Description
token string Yes JWT token for ThingsBoard authentication
customerId string Yes ThingsBoard customer UUID (shopping)
tbBaseUrl string No ThingsBoard API base URL (defaults to current origin)
existingSettings object No Existing settings JSON to pre-populate form
onSave function No Callback after successful save with updated settings
onClose function No Callback when modal closes
styles object No Custom style tokens (primaryColor, borderRadius, etc.)
