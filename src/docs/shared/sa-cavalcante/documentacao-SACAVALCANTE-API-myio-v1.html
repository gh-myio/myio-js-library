📖 Glossário

ThingsBoard Portal MYIO de telemetria — consumo, temperatura, relatórios, dashboards, alarmes e cadastros.
Ingestion Sistema MYIO especializado para API de consumo energético — disponível para ThingsBoard e parceiros.
LUC / SUC Identificador único da loja — ex.: SCMA0Q0101 (Mestre Álvaro).
Device ID UUID do device no ThingsBoard.
Customer ID UUID da Unidade (Shopping) no ThingsBoard.
IngestionId Identificador do medidor no ecossistema Data Ingestion — chave de ligação entre os dois sistemas.
SERVER_SCOPE Escopo de atributos do device com metadados: deviceId (LUC) e ingestionId.
Granularity Precisão de agregação de consumo: 1d (diário) ou 1h (horário).
Grupo Sá Cav. Organização com 7 unidades: Mestre Álvaro, Moxuara, Mont Serrat, Shopping da Ilha, Metrópole Pará, Rio Poty e Head Office.

Autenticação e Listagem de Devices — Grupo Sá Cavalcante
Seção 2.1 – 2.3 ThingsBoard

🔐 2.1 Autenticação — POST /api/auth/login

Passo inicial: obter o token JWT utilizado em todas as chamadas seguintes via header Authorization: Bearer <token>.

POST dashboard.myio-bas.com/api/auth/login
// Body (Content-Type: application/json) { "username": "usuario@sacavalcante.com.br", "password": "••••••••" } // Resposta 200 OK { "token": "eyJhbGciOiJIUzI1NiIsInR5...", "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5..." } // Usar em todas as chamadas seguintes: Authorization: Bearer <token>

📋 2.2 Listagem de Devices por Unidade

Exemplo para o Shopping da Ilha. O mesmo endpoint funciona para qualquer das 7 unidades — basta substituir o customerId.

GET dashboard.myio-bas.com/api/customer/{customerId}/deviceInfos?pageSize=50&page=0
// Exemplo — Shopping da Ilha customer_id: 209424d0-b04f-11f0-9722-210aa9448abc // Campos relevantes por device na resposta { "data": [ { "label": "3F SCSDI102", "id": { "id": "23347230-b04f-11f0-9722-210aa9448abc" }, "customerId": { "id": "209424d0-b04f-11f0-9722-210aa9448abc" } } ], "totalElements": 421, "hasNext": true }

📁 2.3 Mapa Inicial de Devices

Estrutura montada a partir de /deviceInfos. Para cada device:

// Para cada device retornado { label: "3F SCSDI102", deviceUuid: "23347230-b04f-11f0-...", customerUuid: "209424d0-b04f-11f0-...", customerName: "Shopping da Ilha" } // Repetir para todas as 7 unidades

📋 Parâmetros de Listagem

customerId UUID da unidade (Shopping). Ver IDs na Pág. 2. obrigatório
pageSize Resultados por página (ex.: 50) obrigatório
page Índice da página (começa em 0). Iterar até hasNext === false. obrigatório
textSearch Filtro por nome/label do device opcional

Seção 2.4 – 2.5 ThingsBoard

🔍 2.4 Atributos SERVER_SCOPE por Device

Para cada device do mapa inicial, buscar os metadados que contêm o LUC (deviceId/identifier) e o ingestionId:

GET dashboard.myio-bas.com/api/plugins/telemetry/DEVICE/{deviceUuid}/values/attributes/SERVER_SCOPE
// Resposta — array de atributos [ { "key": "deviceId", "value": "SCMA0Q0101" }, { "key": "identifier", "value": "SCMA0Q0101" }, { "key": "ingestionId", "value": "462caf1a-e156-4aa2-..." } ]
deviceId Código LUC da loja. Usar como lucCode no mapa final.
identifier Alternativo ao deviceId como LUC, quando o campo deviceId não estiver presente.
ingestionId UUID do medidor no Data Ingestion. Chave de ligação entre os dois sistemas.

⚠️ Devices ainda sem ingestionId sincronizado não possuem vínculo no Data Ingestion. O LUC permanece disponível via ThingsBoard.

ℹ️ Regras de Enriquecimento

Usar deviceId como lucCode quando presente; caso contrário, usar identifier.
O ingestionId é a chave de ligação entre ThingsBoard e Data Ingestion API.
Devices sem ingestionId ainda não foram sincronizados com o ecossistema de consumo.
Repetir o processo para cada uma das 7 unidades do Grupo Sá Cavalcante.

Seção 2.7 Data Ingestion

🔐 2.7 Autenticação — Data Ingestion API

A Data Ingestion API possui autenticação própria. Token separado do ThingsBoard.

POST api.data.apps.myio-bas.com/api/v1/auth
// Body (Content-Type: application/json) { "clientId": "sacavalcante-client-id", "clientSecret": "••••••••••••••••" } // Resposta 200 OK { "token": "eyJhbGciOiJIUzI1NiIsInR5...", "expiresIn": 3600 } // Usar em todas as chamadas de Ingestion: Authorization: Bearer <token>

📋 Parâmetros — Consulta de Totais

customer_id ID do cliente no Data Ingestion (≠ ThingsBoard UUID). Ex. Shopping da Ilha: 46ac29c5-dea0-4578-b34e-277b55d00fd7 obrigatório
startTime Início ISO-8601 com fuso. Ex.: 2026-04-01T00:00:00-03:00 (URL encoded) obrigatório
endTime Fim ISO-8601 com fuso. Ex.: 2026-04-30T23:59:59-03:00 (URL encoded) obrigatório
deep Quando 1, busca hierárquica (devices em assets abaixo do customer) opcional
groupIds UUID do grupo de devices para filtrar o resultado — ver IDs fixos abaixo opcional

⚠️ Importante — Filtro por Grupo (groupIds)

Adicione groupIds=<uuid> ao endpoint /total (energy ou water) para restringir o resultado a um tipo específico de medidor:

⚡ Energia
Entrada de Energia — medidores de entrada de energia elétrica
groupIds=f431d17a-ec11-45e0-b92c-83a0d3b6d942
💧 Água
Entrada de Água — hidrômetros de entrada
groupIds=510af6ca-74cb-4350-a9bc-d38fc2d8e3c4
GET api.data.apps.myio-bas.com/api/v1/telemetry/customers/{customer_id}/energy/total?groupIds=f431d17a-ec11-45e0-b92c-83a0d3b6d942

Seção 2.8 – 2.9 Data Ingestion

📈 2.8 Consumo de Energia — Totais por Unidade

Exemplo para o Shopping da Ilha. Retorna consumo consolidado de todos os devices no período.

GET api.data.apps.myio-bas.com/api/v1/telemetry/customers/{customer_id}/energy/total
// Exemplo — Shopping da Ilha customer_id: 46ac29c5-dea0-4578-b34e-277b55d00fd7 startTime: 2026-04-01T00:00:00-03:00 endTime: 2026-04-30T23:59:59-03:00 deep: 1 // Resposta — TotalResponse { "id": "46ac29c5-dea0-...", "label": "Shopping da Ilha", "total_kwh": 87340.2, "startTime": "2026-04-01T00:00:00-03:00", "endTime": "2026-04-30T23:59:59-03:00" }

🔗 Customer IDs — Data Ingestion

IDs do cliente no ecossistema de Ingestion (≠ ThingsBoard UUIDs):

⚠️ Os IDs de Ingestion das demais unidades (Mestre Álvaro, Moxuara, Mont Serrat, Metrópole Pará, Rio Poty) devem ser obtidos junto à equipe MYIO na etapa de onboarding de cada shopping.

🔬 2.9 Consumo Granular por Device

Usa o ingestionId do SERVER_SCOPE para dados individuais por medidor.

GET api.data.apps.myio-bas.com/api/v1/telemetry/devices/{device_id}/energy
// 3F SCSDI102 — Shopping da Ilha device_id: 462caf1a-e156-4aa2-8534-8935ba878256 startTime: 2026-04-01T00:00:00-03:00 endTime: 2026-04-30T23:59:59-03:00 granularity: 1d // ou 1h // Resposta — TelemetryResponse [{ "id": "462caf1a-e156-...", "label": "3F SCSDI102", "readings": [ { "bucket": "2026-04-01", "total_kwh": 198.4 }, { "bucket": "2026-04-02", "total_kwh": 204.7 } ] }]

⚙️ Granularidades disponíveis

1dpor dia — relatórios mensais e tendência
1hpor hora — diagnóstico intraday e ponta de demanda

🔄 2.10 Fluxo Completo — ThingsBoard + Data Ingestion
1
Autenticar no ThingsBoard via POST /api/auth/login → JWT.
