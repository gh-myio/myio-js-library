📖 Glossário

Data Ingestion Sistema MYIO especializado para consulta de consumo energético e hídrico — disponível para parceiros via API REST autenticada.
customer_id Identificador da Unidade (Shopping) no ecossistema Data Ingestion. Obtido pela equipe MYIO no onboarding.
device_id Identificador do medidor (device) no ecossistema Data Ingestion — usado para consultas granulares por equipamento.
clientId Identificador da chave de acesso API. Cada unidade possui sua própria chave; o Grupo Soul Malls possui uma chave com visibilidade de todas as unidades.
clientSecret Segredo da chave de acesso — confidencial. Fornecido pela equipe MYIO no onboarding de cada unidade.
Granularity Precisão de agregação de consumo: 1d (diário) ou 1h (horário).
groupIds UUID do grupo de devices — filtra o resultado a um tipo específico (ex.: apenas hidrômetros de entrada ou medidores de energia de entrada).
Grupo Soul Malls Holding com 4 unidades: Ilha Plaza, Plaza Macaé, West Plaza e Praia da Costa — cada uma com chave de acesso própria.

Seção 2.1 – 2.2 Data Ingestion

🔐 2.1 Autenticação — POST /api/v1/auth

Passo inicial: obter o token JWT utilizado em todas as chamadas seguintes. Usar clientId e clientSecret da chave de acesso desejada (por unidade ou Grupo).

POST api.data.apps.myio-bas.com/api/v1/auth
// Body (Content-Type: application/json) { "clientId": "<clientId>", "clientSecret": "••••••••••••••••" } // Resposta 200 OK { "token": "eyJhbGciOiJIUzI1NiIsInR5...", "expiresIn": 3600 } // Usar em todas as chamadas seguintes: Authorization: Bearer <token>
A chave Grupo Soul Malls autentica uma vez e concede acesso às 4 unidades.
Chaves por unidade restringem o acesso ao customer_id correspondente.
Token válido por 3600 segundos (1h) — renovar antes do vencimento.

📋 2.2 Parâmetros Comuns

customer_id ID da unidade no Data Ingestion — ver Pág. 2. Necessário nos endpoints de consumo por unidade. obrigatório
startTime Início do período ISO-8601 com fuso horário. Ex.: 2026-04-01T00:00:00-03:00 (URL encoded) obrigatório
endTime Fim do período ISO-8601 com fuso horário. Ex.: 2026-04-30T23:59:59-03:00 (URL encoded) obrigatório
granularity 1h (padrão) ou 1d — apenas nos endpoints de série temporal, não no /total opcional
deep Quando 1, inclui entidades filhas hierarquicamente (devices em assets abaixo do customer) opcional
page / pageSize Paginação: base 1, padrão 1 · itens por página 1–1000, padrão 100 opcional
groupIds UUID do grupo de devices — restringe resultado por tipo de medidor. Ver Pág. 4. opcional

Seção 2.3 Data Ingestion

⚠️ 2.3 Importante — Filtro por Grupo (groupIds)

Adicione groupIds=<uuid> ao endpoint /total (energy ou water) para restringir o resultado a um tipo específico de medidor. Sem este parâmetro, o total consolida todos os devices do customer.

⚡ Energia
Entrada de Energia — medidores de entrada de energia elétrica
groupIds=f431d17a-ec11-45e0-b92c-83a0d3b6d942
💧 Água
Entrada de Água — hidrômetros de entrada
groupIds=510af6ca-74cb-4350-a9bc-d38fc2d8e3c4
GET api.data.apps.myio-bas.com/api/v1/telemetry/customers/{customer_id}/energy/total?groupIds=f431d17a-ec11-45e0-b92c-83a0d3b6d942
GET api.data.apps.myio-bas.com/api/v1/telemetry/customers/{customer_id}/water/total?groupIds=510af6ca-74cb-4350-a9bc-d38fc2d8e3c4

📋 IDs de Grupo — Referência Rápida

Tipo Descrição groupId

ℹ️ Regras de Uso do Filtro

Sem groupIds, o endpoint /total consolida todos os devices do customer — incluindo lojas, áreas comuns e outros medidores.
Use groupIds de Energia para obter o consumo exclusivo dos medidores de entrada elétrica.
Use groupIds de Água para obter o consumo exclusivo dos hidrômetros de entrada.
O parâmetro funciona da mesma forma para todos os customer_ids do Grupo Soul Malls.

Consumo Total por Unidade e Granular por Device
Seção 2.4 – 2.5 Data Ingestion

📈 2.4 Consumo de Energia — Totais por Unidade

Retorna consumo consolidado de todos os devices do customer no período. Adicione groupIds para filtrar por tipo de medidor (ver Pág. 4).

GET api.data.apps.myio-bas.com/api/v1/telemetry/customers/{customer_id}/energy/total
// Exemplo — Ilha Plaza customer_id: <customer_id-ilha-plaza> startTime: 2026-04-01T00:00:00-03:00 endTime: 2026-04-30T23:59:59-03:00 deep: 1 // Resposta — TotalResponse { "id": "<customer_id-ilha-plaza>", "label": "Ilha Plaza", "total_kwh": 54820.0, "startTime": "2026-04-01T00:00:00-03:00", "endTime": "2026-04-30T23:59:59-03:00" }

O mesmo endpoint funciona para todas as 4 unidades — basta substituir o customer_id.

🔬 2.5 Consumo Granular por Device

Usa o device_id do medidor para dados individuais com granularidade temporal. O device_id é fornecido pela equipe MYIO ou obtido via listagem de devices do customer.

GET api.data.apps.myio-bas.com/api/v1/telemetry/devices/{device_id}/energy
// Exemplo — medidor de entrada Ilha Plaza device_id: <device_id> startTime: 2026-04-01T00:00:00-03:00 endTime: 2026-04-30T23:59:59-03:00 granularity: 1d // ou 1h // Resposta — TelemetryResponse [{ "id": "<device_id>", "label": "Entrada Geral", "readings": [ { "bucket": "2026-04-01", "total_kwh": 1820.4 }, { "bucket": "2026-04-02", "total_kwh": 1794.1 } ] }]

⚙️ Granularidades disponíveis

1dpor dia — relatórios mensais e tendência
1hpor hora — diagnóstico intraday e ponta de demanda

🔄 2.6 Fluxo Completo — Data Ingestion API
1
Autenticar via POST /api/v1/auth com o clientId e clientSecret desejados (por unidade ou Grupo) → JWT.
2
Consultar consumo consolidado por unidade via GET /api/v1/telemetry/customers/{customer_id}/energy/total — adicionar groupIds para filtrar por tipo de medidor.
3
Para água, usar o endpoint equivalente: GET /api/v1/telemetry/customers/{customer_id}/water/total com groupIds de hidrômetros.
4
Para consumo granular por medidor individual: GET /api/v1/telemetry/devices/{device_id}/energy?granularity=1d
5
Com a chave do Grupo Soul Malls, o mesmo token autentica consultas para todas as 4 unidades sem re-autenticação.
